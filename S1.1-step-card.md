# S1.1 step card — site split (do in this order; ~3 min downtime at step 4)
1. github.com → New repository → name `service-break-site` → PUBLIC →
   create (empty, no README init).
2. Upload the bundle: "uploading an existing file" → drag the CONTENTS
   of the `service-break-site` folder (index.html at top level, assets/,
   privacy/, CNAME, manifest.md5, everything) → commit.
3. New repo → Settings → Pages → Source: Deploy from a branch → main /
   root → Save. Wait for the green "Your site is live" (1–2 min). It
   serves at peterlitton.github.io/service-break-site for now.
4. THE SWAP (site blinks here): OLD repo `service-break` → Settings →
   Pages → remove the custom domain, then set Source: None. Immediately:
   NEW repo → Settings → Pages → Custom domain: `service-break.com` →
   Save → check "Enforce HTTPS" when it appears (cert may take a few
   minutes; the CNAME file in the bundle keeps this sticky).
5. Tell A.3 "swapped" — probes run: apex 200, privacy 200, byte-compare
   vs bundle, sitemap/robots, HTTPS. PASS unblocks S1.2 (the flip).
Note: DNS at the registrar does NOT change — it already points at
GitHub Pages; only the repo binding moves.
